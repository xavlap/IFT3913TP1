L'utilisation d'ergon pour trouver les classes divines à un sens. Nous trouvons les classes qui font le plus référence à d'autres classes et/ou qui sont le plus référencées par d'autres classes et qui ont le plus de lignes de code. Ces caractéristiques sont typiques des classes divines donc, si on regarde parmis les classes données par ergon, il est très probable qu'on se rendre compte qu'il y a place à amélioration par rapport aux classes divines.
Pour améliorer ergon nous pourrions premièrement ordonner les choses, se qui est mieux pour nous les humains, mais qui n'améliore pas vraiment egon en tant que tel. Nous pourrions aussi compter le nombre de méthode qui sont indépendante l'une de l'autre, c'est-à-dire que si une methode en utilise une autre on ne les compte que pour une. Ceci serait bien puisque, s'il y a beaucoup de méthode utilisable, c'est probablement qu'elles sont utilisées et si on ajoute nos deux informations qu'on a déjà, nous pouvons trouver les classes divines encore mieux. Si une classe est référencer dans chacunes des autres classes mais toujours pour la même méthode, on ne peut pas vraiment dire que c'est une classe divines, voila pourquoi je propose cette amélioration.
